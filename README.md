Основной функционал  
Для участников:  
• Регистрация — пользователь вводит своё имя и фамилию, бот проверяет членство в группе  
• Просмотр расписания — доступны ближайшие тренировки из расписания  
• Запись на тренировку — через кнопку в групповом чате или личном сообщении  
• Отмена записи — возможность отменить свою запись до начала тренировки  

Для администраторов (тренеров):  
• Создание тренировки — из расписания (слоты) или с произвольными датой/временем/локацией  
• Назначение лимита мест — ограничение количества участников  
• Отмена тренировки — автоматическое уведомление всех записавшихся  
• Просмотр списка участников — в личных сообщениях видны все записавшиеся, в группе — только участники чата  

Автоматика:  
• Автозакрытие тренировок — через 1.5 часа после начала тренировка и записи помечаются как "прошедшие"  
• Логирование событий — все действия (запись, отмена, создание тренировки) сохраняются в таблице events  

Технологический стек  
Язык: Python 3.10+  
Библиотеки: aiogram 3.x, mysql-connector-python, python-dotenv  
База данных: MySQL 8.0  
Платформа: Telegram Bot API  


Сценарии:  
1. Добавление бота в группу  
Администратор добавляет бота в Telegram-группу  
Бот автоматически создаёт запись в participant_groups  
Группа становится доступна для тренировок  

2. Регистрация пользователя  
Пользователь пишет боту /start  
Бот проверяет членство в группе  
Запрашивает имя и фамилию  
Сохраняет пользователя в БД со статусом "участник"  

3. Создание тренировки (из расписания)  
Администратор открывает меню выбора тренировки  
Выбирает свободный слот из расписания  
Подтверждает создание  
Бот отправляет сообщение в группу с кнопкой "Записаться"  
Сохраняет сообщение для последующего обновления  

4. Создание тренировки (произвольная)  
Администратор выбирает "Другое" в меню  
Выбирает дату → вводит время → выбирает локацию  
Выбирает группу (или одну группу, если show_groups=False)  
Указывает лимит мест (или создаёт без лимита)  
Бот создаёт тренировку и отправляет сообщение в группу  

5. Запись на тренировку
Участник нажимает "Записаться" под сообщением в группе  
Бот проверяет: наличие мест, активность тренировки, отсутствие дублей  
Добавляет запись в БД  
Обновляет сообщение в группе и личное сообщение администратора  

6. Отмена тренировки
Администратор нажимает "Отменить" в личном сообщении  
Подтверждает отмену  
Бот:  
Меняет статус тренировки на "отменена"  
Отменяет все записи  
Удаляет сообщение в группе  
Отправляет уведомление с упоминанием всех записавшихся  

7. Автозакрытие
Фоновый процесс проверяет активные тренировки  
Если тренировка завершилась:  
Меняет статус на "прошла"  
Меняет статус всех записей на "прошла"  
Обновляет сообщения  

База данных:
users — пользователи (имя, telegram_id, статус, группа)  
participant_groups — группы участников (привязаны к Telegram-чатам)  
trainings — тренировки (дата/время, локация, статус)  
registrations — записи на тренировки (связь user-training)  
events — журнал событий (аудит)  
messages — связь сообщений с тренировками для обновления  
slots — расписание (день, время, локация)  
locations — локации тренировок  
slots_groups / trainings_groups — привязка групп к слотам/тренировкам

Статусы:
• Пользователи: участник, участник-админ, тренер, неактивен  
• Тренировки: назначена, отменена, прошла  
• Записи: активна, отменена участником, отменена тренером, прошла
